# Руководство по участию в проекте

## Как начать работу
1. Сделать fork репозитория
2. Создать новую ветку: feature/название-задачи
3. Внести изменения в код
4. Сделать commit с понятным описанием
5. Отправить Pull Request

## Правила работы
- Один commit — одна логическая задача
- Названия веток: feature/..., bugfix/...
- Комментарии к коммитам писать на русском языке
- Перед отправкой проверять работоспособность кода

## Для участников
- Соблюдать структуру проекта
- Не изменять чужой код без необходимости
- Обсуждать изменения через Issues
